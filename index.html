Typeface — Lexend
Lexend is the sole typeface across all platforms. Designed specifically for reading proficiency and legibility — ideal for both seniors and data-heavy admin views. Use weights 400, 500, 600, 700 only.
Aa Bb Cc 123
700 BoldPage titles, hero headings
Section Heading
600 SemiBoldSection titles, card headers
Body text reads clearly even in longer paragraphs. Lexend's letterforms reduce crowding and visual stress for all readers.
400 RegularBody copy, descriptions, table content
Section Label / Overline
700 Bold · Uppercase · +0.08em trackingCategory labels, section overlines
Weights to use: 400, 500, 600, 700 only. Do not use 300 (Light) — it fails legibility at small sizes.

Guidelines
Do's & Don'ts
✓ Colours
Use only the named palette colours
Pair semantic colours with their light tint for badge backgrounds
Use #3D72E8 as the one key action colour per screen
✗ Colours
Don't invent new blues — only #3D72E8 and #2D5EC7
Don't use pure black #000000 for text — use #1A2233
Don't use colour alone to convey status — pair with label
✓ Typography
Use Lexend for all text, all platforms
Follow the canonical size table per platform
Increase font size (not weight) for senior contexts
✗ Typography
Don't use any other typeface
Don't use weight 300 (Light)
Don't use the same sizes for senior and staff screens
✓ Icons
Use Phosphor Icons (fill variant) as primary library
Pair icons with text labels on senior and kiosk screens
Use coloured tint containers for feature category icons
✗ Icons
Don't use emoji as icons in any UI screen
Don't use icon-only navigation on senior screens
Don't mix icon libraries — stick to Phosphor
✓ Senior & Kiosk UX
Keep to 1 primary CTA per screen
Minimum 48×48px touch targets on senior, 64×64px on kiosk
Use illustrations to explain, not just text
✗ Senior & Kiosk UX
Don't use body text under 16px on senior interfaces
Don't show more than 2–3 items per scroll on kiosk
Don't use hover-only interactions — all must be touch

CaritaHub Brand Prompt · v1.1
## CaritaHub Brand Guidelines

You are designing a UI screen for **CaritaHub**, a care management platform. Follow these brand guidelines exactly.

---

### PLATFORM & USER CONTEXT
Before designing, confirm which of these applies to this screen:
- **Web Admin** — desktop browser, staff / counsellors / admins
- **Mobile — Staff / NOK / Volunteers** — iOS or Android, normal adults
- **Mobile — Senior** — iOS or Android, elderly users (larger text, simpler layout)
- **Tablet — Staff / NOK** — iPad or Android tablet, normal adults
- **Tablet — Senior** — iPad or Android tablet, elderly users
- **Kiosk** — public touch screen, walk-in seniors (very large text, large touch targets)

---

### COLOURS
Use only these values. Do not invent new shades.

| Token | Hex | Use |
|---|---|---|
| Primary | #3D72E8 | Buttons, links, active states, key UI |
| Primary Dark | #2D5EC7 | Hover / pressed states |
| Primary Light | #EBF2FF | Active nav bg, chip bg, row highlight |
| Accent | #E07820 | Rewards, points, highlights |
| Accent Light | #FFF3E8 | Accent badge backgrounds |
| Success | #2DAF7B | Confirmed, completed |
| Success Light | #E6F9F2 | Success badge bg |
| Danger | #E84B3D | Errors, alerts, destructive actions |
| Danger Light | #FEF0EF | Danger badge bg |
| Purple | #7B5EA7 | AI features, Ava, engagement |
| Purple Light | #F0EBF8 | AI / Ava badge bg |
| Teal | #1A9FAF | Health data, clinical metrics |
| Teal Light | #E5F6F8 | Health badge bg |
| Text | #1A2233 | All body text and headings |
| Muted | #6B7A99 | Supporting text, labels, placeholders |
| Background | #F4F6FB | Page / screen background |
| Border | #DDE3F0 | Card borders, dividers, input borders |
| White | #FFFFFF | Card surfaces, inputs, modals |

Rules:
- Never use pure black (#000000) for text.
- Never use colour alone to convey status — always pair with a label or icon.
- One Primary (#3D72E8) button maximum per screen view.

---

### SPACING
All spacing is multiples of 4px.
- xs: 4px — icon gaps, tight chip padding
- sm: 8px — inline gaps, badge padding
- md: 12px — card inner gap, list item gap
- base: 16px — card padding, section gap (mobile)
- lg: 20px — card padding (desktop), column gap
- xl: 24px — section gap, page horizontal padding (mobile)
- 2xl: 32px — section gap (desktop)
- 3xl: 48px — page top padding

Page horizontal padding: Web 48–56px · Tablet 24px · Mobile 16px · Kiosk 20px

---

### BORDER RADIUS & SHADOW
- Badges / chips: 20px (pill)
- Inputs, small elements: 8–10px
- Cards (Admin & Staff platforms): 14px
- Cards (Senior & Kiosk platforms): 18–20px
- Avatars / profile photos: 50%

Shadows use blue-tinted values only:
- Default (cards): 0 2px 14px rgba(61,114,232,0.08)
- Medium (modals, dropdowns): 0 6px 24px rgba(61,114,232,0.13)
- Large (bottom sheets): 0 8px 32px rgba(61,114,232,0.16)

Never use sharp-cornered (0px radius) containers anywhere.

---

### NAVIGATION PATTERNS
- **Web Admin & Tablet Staff**: Left sidebar, 240px wide. Active item = Primary Light bg + Primary Blue text + filled icon.
- **Mobile — Staff / NOK**: Bottom tab bar, max 5 items, 66px height, 12px/600 labels, 22px icons. Active item = Primary Blue text + 3px top border (32px wide, rounded bottom corners). Inactive = muted.
- **Mobile — Senior**: Bottom tab bar, max 4 items, 64px height, 13px labels, 26–28px icons.
- **Kiosk**: Top row of icon buttons (64×64px min containers, 14px radius). Active = Primary Blue (#3D72E8) background + white icon. Inactive = white background + dark icon + Border border.

---

### BUTTONS
- Font: Lexend 600
- One Primary button maximum per screen
- Variants: Primary (blue fill), Secondary (Primary Light fill), Outline (blue border), Ghost (no bg), Danger (red fill)
- Size scales with platform: 12px small (admin) → 14px base (staff mobile) → 16px large (senior mobile) → 18px XL (kiosk)
- Border radius matches the platform's card radius

---

### BADGES & STATUS TAGS
- Shape: pill (border-radius: 20px)
- Always pair the semantic colour with its paired light tint as background
- Example: Confirmed = #2DAF7B text on #E6F9F2 bg; High Risk = #E84B3D text on #FEF0EF bg

---

### ICONS
Use **Phosphor Icons** (fill variant) as the icon library — flat filled shapes, no outline strokes visible on the shapes themselves.
- Web (CDN): `` then ``
- Figma: search "Phosphor Icons" in Figma Community
- SVG download: phosphoricons.com

Icon sizes:
- Inline with text: 16px
- Button icon (admin): 16–18px
- Nav icon (staff mobile): 20–22px
- Nav icon (senior mobile): 26–28px
- Kiosk nav (inside 64px container): 30–32px
- Feature / hero icon: 48px+

Icon colour rules:
- Active / CTA: white icon on Primary Blue container
- Inactive: muted (#6B7A99) filled icon
- Category feature icon: Primary Blue filled icon on Primary Light (#EBF2FF) container
- Always pair icons with a visible text label on senior and kiosk screens

Do NOT use emoji as icons in any UI screen.

---

### IMAGERY & ILLUSTRATION
**Photography** (activity cards, event banners): warm tones, diverse active seniors, bright settings, joyful. Avoid clinical, isolated, or dark imagery.

**Illustrations** (empty states, onboarding, feature explainers): flat vector, bold outline 2–3px, rounded shapes, colours from the brand palette, friendly characters with white/grey hair for senior context.
- Source: unDraw (undraw.co) or Storyset (storyset.com) if image generation is unavailable.

**Avatars**: circle (50% radius), show initials on gradient background when no photo.

---

### PLATFORM-SPECIFIC RULES

**Senior (mobile & tablet):**
- Never use body text under 16px
- Always label icons with text
- Max 1 primary CTA per screen
- Min touch target 48×48px (mobile), 52×52px (tablet)
- Use flat illustrations throughout

**Kiosk:**
- Min touch target 64×64px
- Max 2–3 actions visible per scroll
- Show a session countdown timer (top-right corner)
- All interactions must be touch-accessible — no hover-only states
- Navigation active state: Primary Blue (#3D72E8) — not red

**Web Admin:**
- High information density is acceptable
- Use tables, stat cards, and feeds as primary layouts
- Sidebar width: 240px

---

### UX PATTERNS — MOBILE STAFF & NOK

**Screen structure (top to bottom):**
1. Status bar (44px, white bg, 15px/700 time)
2. Senior context bar — shows active senior name (10px caption + 14px/700 name) with switch button (12px/600, 20px radius, 1.5px border)
3. Page header — white bg, 16px 20px padding, 24px/700 title + 11px/muted subtitle, icon buttons (36×36px, 50% radius, --bg fill) on right, 1px border-bottom
4. Scrollable content — padding: 20px 18px 100px, gap: 24px between sections
5. Bottom nav — 66px height, white bg, 1px border-top

**Page header icon buttons:**
- Size: 36×36px, border-radius: 50%, background: --bg
